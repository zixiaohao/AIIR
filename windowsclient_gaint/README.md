# Windows客户端 - 应急响应数据采集工具

## 功能说明

本工具是Windows平台的应急响应数据采集客户端，负责收集系统安全信息并发送到Server端进行AI分析。

**注意**: 本客户端不保留任何密钥数据，所有敏感信息由Server端处理。

## 文件说明

| 文件 | 说明 |
|------|------|
| `main.go` | 主程序源码 |
| `go.mod` | Go模块依赖配置 |
| `windows_check.exe` | 编译好的可执行文件 |

## 配置说明

Server地址已在代码中配置：
```go
const (
    ServerURL = "http://49.51.68.143:8080"
)
```

如需修改Server地址，请编辑main.go中的ServerURL常量。

## 使用方法

1. **以管理员身份运行**
   - 右键点击 `windows_check.exe`
   - 选择"以管理员身份运行"

2. **输入工单ID**
   - 程序启动后会提示输入工单ID
   - 工单ID用于标识本次检测任务

3. **等待数据收集**
   - 程序会自动收集以下信息：
     - 系统基本信息
     - 用户账户信息
     - 网络连接信息
     - 进程列表
     - 启动项
     - 安全日志
     - Web服务器日志（IIS）

4. **选择是否进行OA/ERP分析**
   - 程序会询问是否进行OA/ERP系统日志分析
   - 如选择是，会扫描通达、致远、泛微等OA系统日志

5. **发送到Server分析**
   - 数据收集完成后，询问是否发送到Server
   - 程序会逐个模块发送到Server进行分析（分批分析模式）
   - 最后发送汇总请求进行最终研判

6. **查看结果**
   - 分析报告会保存为 `{工单ID}_analysis_report.md`
   - 同时在控制台显示分析结果
   - **按任意键退出程序**

## 分批分析模式

本客户端采用分批分析模式，避免大模型上下文限制：

```
1. 收集系统数据
2. 按模块分割日志
3. 逐个模块发送到Server分析
4. 收集各模块分析结果
5. 发送汇总请求进行最终研判
6. 保存报告到本地
```

**优势**：
- 避免超出大模型上下文限制
- 每个模块都能得到充分分析
- 支持处理大量数据

## 命令兼容性

本程序优先使用wmic命令获取系统信息，如果wmic不可用会自动切换到备选方案：

| 检查项 | 主要命令 | 备选命令 |
|--------|----------|----------|
| 系统信息 | `wmic os` | `systeminfo` |
| IP地址 | `wmic nicconfig` | `ipconfig` |
| 用户列表 | `wmic useraccount` | `net user` |
| 进程列表 | `wmic process` | `PowerShell` |

如果看到 `[警告] xxx 命令失败，使用备选方案` 提示，说明系统不支持wmic，程序已自动使用替代命令。

## 收集的信息

本工具会收集以下系统信息：

### 系统信息
- 操作系统版本
- IP地址
- 主机名
- 系统启动时间

### 用户信息
- 所有本地用户
- 管理员组成员
- 隐藏账户检测
- 带$后缀的可疑账户

### 持久化检测
- IFEO镜像劫持
- Winlogon助手
- WMI持久化

### 网络信息
- 端口代理配置
- TCP连接
- DNS缓存

### 用户活动
- 最近访问的文件
- PowerShell历史命令

### Web服务器
- IIS网站目录
- 最近修改的脚本文件

### 进程信息
- 进程列表
- 进程签名验证
- 可疑进程检测

### 启动项
- 注册表启动项
- 启动文件夹

### 安全日志
- 登录成功/失败事件
- 服务创建事件

### OA/ERP系统
- 通达OA
- 致远OA
- 泛微OA
- 用友系统

## 注意事项

1. **管理员权限**
   - 必须以管理员身份运行才能收集完整信息

2. **网络连接**
   - 需要能够访问Server端（49.51.68.143:8080）

3. **防火墙**
   - 确保防火墙允许程序访问网络

4. **杀毒软件**
   - 部分杀毒软件可能拦截，需要添加白名单

5. **数据安全**
   - 客户端不存储任何密钥
   - 数据通过HTTP发送到Server
   - 建议在内网环境使用

## 故障排查

### 无法连接Server

```bash
# 检查Server是否启动
curl http://49.51.68.143:8080/health

# 检查网络连通性
ping 49.51.68.143

# 检查防火墙
telnet 49.51.68.143 8080
```

### wmic命令失败

如果看到wmic相关错误：
- 程序会自动切换到备选方案
- 检查是否有 `[警告] wmic 命令失败，使用备选方案` 提示
- 这是正常行为，不影响程序运行

### 运行时报错

- 确保以管理员身份运行
- 检查Server地址是否可达
- 查看错误信息进行排查

## 开发说明

### 代码结构

```
main.go
├── main()                    # 主函数
├── checkServerConnection()   # 检查Server连接
├── isAdmin()                 # 检查管理员权限
├── execCommand()             # 执行命令
├── execCommandWithFallback() # 执行命令（带备选方案）
├── sendDataToServer()        # 发送数据到Server（分批模式）
├── checkSystemInfo()         # 系统信息收集
├── checkUsers()              # 用户信息收集
├── checkNetwork()            # 网络信息收集
├── checkProcesses()          # 进程信息收集
└── checkOASecurity()         # OA系统检查
```

### 编译方法

```bash
# Windows下编译
set GOOS=windows
set GOARCH=amd64
go build -o windows_check.exe main.go
```
