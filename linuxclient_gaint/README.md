# Linux客户端 - 应急响应数据采集工具（gaint版本）

## 功能说明

本工具是Linux平台的应急响应数据采集客户端，负责收集系统安全信息并发送到Server端进行AI分析。这是gaint版本，采用一次性发送模式。

**注意**: 本客户端不保留任何密钥数据，所有敏感信息由Server端处理。

## 文件说明

| 文件 | 说明 |
|------|------|
| `client_gaint.sh` | 主程序脚本（一次性发送版本） |
| `busybox` | 工具箱（可选，提供兼容性命令） |
| `vuln` | 漏洞扫描工具（可选） |

## 配置说明

**重要**: 本客户端不再硬编码Server地址，用户需要在运行时通过命令行参数指定Server地址。

## 使用方法

### 基本用法

1. **以root权限运行**

2. **指定Server地址运行**
   ```bash
   ./client_gaint.sh -s http://YOUR_SERVER_IP:8000
   ```
   
   或使用长参数：
   ```bash
   ./client_gaint.sh --server http://YOUR_SERVER_IP:8000
   ```

3. **查看帮助信息**
   ```bash
   ./client_gaint.sh -h
   ```

### 运行示例

```bash
# 使用短参数
./client_gaint.sh -s http://192.168.1.100:8000

# 使用长参数
./client_gaint.sh --server http://10.0.0.50:8000

# 查看帮助
./client_gaint.sh -h
```

### 交互式输入

如果不指定Server地址，脚本会提示手动输入：
```bash
./client_gaint.sh
```

脚本会显示：
```
请输入Server地址 (格式: http://IP:端口): 
```

此时输入Server地址（如：`http://192.168.1.100:8000`）即可。

3. **输入工单ID**
   - 脚本启动后会提示输入工单ID
   - 工单ID用于标识本次检测任务

4. **等待数据收集**
   - 脚本会自动收集以下信息：
     - 系统基本信息
     - 用户账户信息
     - 网络连接信息
     - 进程列表
     - 启动项
     - 定时任务
     - 安全日志

5. **发送到Server分析**
   - 数据收集完成后，一次性发送到Server进行分析
   - 适合大模型上下文窗口较大的场景

6. **查看结果**
   - 分析报告会保存为 `{工单ID}_analysis_report.md`
   - 同时在控制台显示分析结果
   - **按任意键退出脚本**

## 一次性发送模式

本客户端采用一次性发送模式，适合大模型上下文窗口较大的场景：

```
1. 收集系统数据
2. 将所有日志打包
3. 一次性发送到Server分析
4. 保存报告到本地
```

**优势**：
- 简单高效，只需一次网络请求
- 适合大模型上下文窗口较大的场景
- 减少网络交互次数

**适用场景**：
- 大模型支持较大上下文窗口（如128K tokens以上）
- 网络延迟较高的环境
- 数据量相对较小的场景

## 收集的信息

本工具会收集以下系统信息：

### 系统信息
- 操作系统版本
- 内核版本
- CPU信息
- 内存信息
- 磁盘空间
- IP地址（内网/公网）
- 主机名

### 用户信息
- 所有本地用户
- 特权用户（UID 0）
- 当前登录用户
- 用户历史命令

### 网络信息
- ARP表项
- 网络连接
- 路由表
- 防火墙规则
- 开放端口

### 进程信息
- 进程列表
- 可疑进程检测

### 自启动项
- init.d服务
- systemd服务

### 定时任务
- 系统crontab
- 用户crontab

### 关键文件
- hosts文件
- SSH密钥文件
- /tmp目录

### 日志信息
- 登录成功记录

## 注意事项

1. **root权限**
   - 必须以root身份运行才能收集完整信息

2. **网络连接**
   - 需要能够访问Server端

3. **curl依赖**
   - 需要系统安装curl命令
   - 如果未安装curl，会跳过AI分析，只保存本地日志

4. **数据安全**
   - 客户端不存储任何密钥
   - 数据通过HTTP发送到Server
   - 建议在内网环境使用

5. **大模型限制**
   - 一次性发送模式适合大模型上下文窗口较大的场景
   - 如果数据量过大，可能超出大模型的上下文限制
   - 此时建议使用分批分析版本（client.sh）

## 故障排查

### 无法连接Server

```bash
# 检查Server是否启动
curl http://YOUR_SERVER_IP:8000/health

# 检查网络连通性
ping YOUR_SERVER_IP

# 检查防火墙
telnet YOUR_SERVER_IP 8000
```

### curl命令不存在

```bash
# Ubuntu/Debian
apt-get install curl

# CentOS/RHEL
yum install curl

# 或使用wget替代（需要修改脚本）
```

### 运行时报错

- 确保以root身份运行
- 检查Server地址是否可达
- 查看错误信息进行排查

### 分析失败（上下文超限）

如果Server返回分析失败，可能是因为数据量过大超出大模型上下文限制：
- 使用分批分析版本（client.sh）
- 或联系管理员配置支持更大上下文的模型

## 离线模式

如果无法连接到Server，脚本会自动进入离线模式：
- 执行本地安全检查
- 保存日志文件到本地
- 提示用户后续处理

## 开发说明

### 代码结构

```
client_gaint.sh
├── show_help()                  # 显示帮助信息
├── 解析命令行参数               # -s/--server, -h/--help
├── 验证Server地址格式
├── 检查root权限
├── run_cmd()                    # 执行命令（兼容busybox）
├── 系统信息收集                 # 10个检查模块
├── 一次性发送到Server           # analyze_full接口
├── 上传文件到Server             # 日志和报告
└── perform_offline_security_check() # 离线安全检查
```

### 依赖工具

- bash
- curl（用于Server通信）
- python3（用于JSON解析，可选）
- busybox（可选，提供兼容性）

## 与分批分析版本的区别

| 特性 | gaint版本（本版本） | 分批分析版本 |
|------|-------------------|-------------|
| 发送方式 | 一次性发送 | 逐个模块发送 |
| 网络请求次数 | 1次 | N+1次（N为模块数） |
| 适用场景 | 大上下文窗口模型 | 标准上下文窗口模型 |
| 数据量限制 | 受大模型上下文限制 | 可处理较大数据量 |
| 复杂度 | 简单 | 相对复杂 |
